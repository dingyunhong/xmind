2.postman断言
	1.Status code:Code is 200   响应状态码是否是200 
		pm.test("Status code is 200", function () {
			pm.response.to.have.status(200);
		});
		
	2.Response time is less then 200ms   响应时间是否低于200ms  
		pm.test("Response time is less than 200ms", function () {
			pm.expect(pm.response.responseTime).to.be.below(200);
		});
		
	3.Response body: Contains string  响应包含某个字符串  
		pm.test("Body matches string", function () {
			pm.expect(pm.response.text()).to.include("string_you_want_to_search");
		});
		
	4.Response body is equal to a string 响应主体等于某个值
		pm.test("Body is correct", function () {
			pm.response.to.have.body("response_body_string");
		});
		
	5.Response body：JSON value check  响应主体中的某个json键的值是否等于某个值（常用：重点）
		pm.test("Your test name", function () {
			var jsonData = pm.response.json();
			pm.expect(jsonData.value).to.eql(100);
		});

4.postman抓取app包

	前言：前台封装参数发送请求http请求到后台服务器获取数据，
		  如果是查询，则会把后台返回的查到的json格式的数据展示在前台供用户查看
		  如果是增删改，则会通过后台对数据库进行修改，且后台同样会返回json格式的数据，
			只是这种是提示类型的json数据（比如：增删改成功），前台根据后台返回的提示信息提示使用是否增删改成功
	所以：
		  既然前后台调通就是因为http请求，那么postman也可以模拟前台发送http请求到后台获取后台数据，而不需要经过前台就可以实现
		  
	
	现在：要实现postman抓取前台发送到后台的请求接口，咋办？？？
		  设置postman代理
		  
		  因为前台发送http请求是通过某个网络走到后台去的，现在postman是这个网络的代理，所以它能拦截请求的地址

	
	如何设置代理：
		1.要求postman工具所在机器和app所在的终端连载同一个wifi中
		2.在postman中设置代理端口
			https://blog.csdn.net/saddyyun/article/details/91440791
			
			1.服务器ip地址：填写自己电脑的ip地址，因为postman在你的电脑上  查看自己的ip地址：cmd  ipconfig
				192.168.0.136
			2.端口号：可以顺便填写
			
		3.在app所在终端设置代理的ip地址及端口号
